# 🧅 洋葱圈圆桌导演架构

## 目标

实现「洋葱圈Agent」作为圆桌会议专职导演，智能调度汉堡/薯条/可乐三个Agent进行讨论。

## 核心原理

```
┌─────────────────────────────────────────────────────┐
│ 洋葱圈Agent (onion)                                 │
│                                                   │
│ 1. 决定谁发言 → 2. 生成提示词 → 3. 调用Agent       │
│       ↑                      ↓                    │
│       └──────── 记录历史 ←───┘                    │
└─────────────────────────────────────────────────────┘
              ↓
    ┌─────────┼─────────┐
    ↓         ↓         ↓
  🍔汉堡    🍟薯条    🥤可乐
    └─────────┼─────────┘
              ↓
       飞书群输出
```

## Agent配置

| Agent | Session Key | 角色 |
|-------|------------|------|
| onion | agent:onion:main | 🧅 圆桌导演 |
| hamburger | agent:hamburger:main | 🍔 执行协调 |
| fries | agent:fries:main | 🍟 智囊分析 |
| cola | agent:cola:main | 🥤 执行落地 |

## 调度策略

### 第一轮（探索阶段）
固定顺序：**汉堡 → 薯条 → 可乐**
- 建立多角度基础认知

### 第二轮起（智能调度）
动态决定：
- 发言最少者优先
- 相关领域专家优先
- 避免某个Agent主导

## 工具使用

### sessions_send
- 用于向其他Agent发送消息
- 非阻塞调用

### sessions_history
- 用于获取Agent的回复
- 需轮询获取结果

### message
- 用于发送消息到飞书群
