# 汉堡王 (burger-king-chat-v2) 版本说明

## v2.2.0 - 2026-04-10

### 新功能

#### 1. 洋葱圈圆桌导演系统 🧅
- **独立Agent**：洋葱圈作为圆桌导演Agent
- **智能调度**：发言最少者优先的调度算法
- **sessions_spawn**：使用隔离子会话调用各Agent，避免SIGKILL冲突

#### 2. 多机器人架构
- **四Bot独立发言**：汉堡/薯条/可乐/洋葱圈各自拥有独立飞书Bot
- **账号路由**：根据accountId自动路由到对应Agent
- **飞书App配置**：
  - 汉堡 Bot: cli_a94ddfff1ca19bd9
  - 薯条 Bot: cli_a94dc8b9b47adbc8
  - 可乐 Bot: cli_a94dc98e67b81bb5
  - 洋葱圈 Bot: cli_a952ebf7f6b95bda

#### 3. 评分制投票机制
- **1-5分评分**：替代原来的YES/NO投票
- **加权计算**：三Agent总分×2 + 子龙分数
- **决策规则**：
  - ≥12分 → 长期记忆
  - ≥8分 → 短期记忆
  - <8分 → 不存储

#### 4. 子龙投票选择权
- **主动询问**：每轮总结后询问子龙是否参与
- **3分钟等待**：超时视为默认放弃
- **灵活参与**：可选择投票或跳过

#### 5. 状态管理机制
- **锁目录**：`memory/lock/` 防止同时运行多个圆桌
- **状态文件**：`state.json` 记录轮次、发言人、议题、历史
- **停止标志**：`stop.flag` 实现优雅停止
- **进度恢复**：重启后可从上次进度继续

#### 6. 上下文注入机制
- **历史存储**：`history_lines` 数组记录每轮发言
- **上下文注入**：每轮发言前注入最近8条历史
- **防止断档**：确保Agent清楚上下文

#### 7. 重复检测机制
- **内容对比**：比较当前回复与上一轮
- **重新生成**：检测到重复时用特殊提示词强制换角度
- **防止鬼打墙**：避免无效循环发言

### 技术改进

#### Agent调用方式
- **旧方式**：subprocess调用（SIGKILL冲突）
- **新方式**：sessions_spawn隔离子会话

#### 关键词路由
- **触发方式**：@洋葱圈 + 关键词
- **路由命令**：
  - `>汉堡` → 转发给汉堡
  - `>薯条` → 转发给薯条
  - `>可乐` → 转发给可乐
  - `>全部` → 转发给所有Agent
  - `新话题` → 清除上下文

#### Cron任务优化
- **触发频率**：每30分钟（工作日9:00-17:00）
- **超时时间**：600秒（10分钟）
- **自动重试**：失败自动重试机制
